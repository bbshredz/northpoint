Five distinct gaps currently limit the IT department's ability to support NACS at its current size ‚Äî and each one compounds as the organization grows. Addressing them now, before the next four facilities come online, is materially less expensive than addressing them after.

Helpdesk Scalability ‚Äî No AI Layer
The helpdesk currently operates as a fully manual function with no formal structure, tiers, or automation. There is no Level 0 routing ‚Äî every ticket requires direct human handling. As the facility count grows, this does not scale without proportional headcount. Implementing AI-assisted L0 triage (auto-routing, common resolutions, self-service) would dramatically reduce human touchpoints and allow the team to focus on higher-order work. This requires a formalized helpdesk function and an owner with the authority to build it.
No L0 automation ¬∑ headcount scales linearly with demand
üîç
Security Visibility Gap ‚Äî No NDR
NACS has endpoint protection (SentinelOne) and email security (Coronet), but no network-layer behavioral detection. Tools like Darktrace, ExtraHop, or similar Network Detection & Response solutions monitor east-west traffic for anomalies that endpoint tools cannot see. As the facility count grows and network complexity increases, the absence of this layer represents an escalating and addressable risk ‚Äî one that sits squarely in Geremia's proposed security scope.
Endpoint + email covered ¬∑ network layer blind ¬∑ no NDR deployed
